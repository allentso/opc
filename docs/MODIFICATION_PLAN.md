## 一、当前问题诊断

### 1.1 截图中可见的核心问题

| 问题 | 具体表现 | 严重度 |
|------|---------|--------|
| **信息密度极低** | 主聊天区几乎空白，只有2条系统消息 | 🔴 致命 |
| **右侧频道栏不可用** | 只显示单个汉字/字母，玩家无法识别频道 | 🔴 致命 |
| **今日策略是假按钮** | StatusBar有chip但点击无响应，制造虚假交互 | 🔴 致命 |
| **工作台入口不明显** | 最重要操作（接单）藏在Tab内，无法形成引导 | 🟡 重要 |
| **战报为纯文本** | 分享卡 MVP 是一段文字，没有截图价值 | 🟡 重要 |
| **频道名称太窄** | 右侧频道列表仅一个字，无法区分内容 | 🔴 致命 |

### 1.2 根本性设计问题

```
现状：玩家看到的是「一个空的聊天界面 + 几条系统通知」
目标：玩家看到的是「一个正在运转的AI公司，发生着真实的对话和事件」

信息密度 ≠ 消息堆砌
需要的是：有层次感、有上下文的活跃对话流
```

## 二、架构级改动

### 2.1 主界面布局重构

**现在（问题）：**
```
[状态栏]
[主聊天区：几乎空白]        [右侧：一列小圆点]
[底部Tab：消息/工作台/指挥/财务]
```

**改后（目标）：**
```
[状态栏 — 公司名 + 今日策略(可点击) + 声誉 + 资金]
[警报横幅 — 仅在冲突/事故时显示，红色]
[左侧频道边栏 72px]  [主聊天区 flex:1]
  — 竖排频道图标         — 频道名头部
  — 彩色头像(区分部门)   — 活跃AI对话
  — 未读红点            — 分层消息气泡
  — 私下频道独立分区     — 事故卡片样式
[底部Tab：消息/工作台/指挥/财务]
[输入栏 — 公开频道可发言]
```

关键改变：
- 右侧小圆点 → 左侧带彩色头像+图标的频道边栏（72px）
- 频道分两组显示：公开频道 / 私下频道（深色分隔）
- 私下频道使用深色 #1C1C23 背景的头像，视觉上有区隔感

### 2.2 今日策略：从假按钮变为真入口

**状态栏 chip** → 变为可点击按钮，点击进入今日策略全屏页面  
**工作台顶部** → 保留策略 banner，也可点击跳转

今日策略页面结构（独立全屏Tab页或底部Sheet）：
```
[提示说明 — 策略影响范围的简要说明]
[策略选项列表 — 每项包含图标、名称、效果描述]
[解锁策略（灰化+锁图标） — 需要特定条件]
[确认按钮]
```

### 2.3 工作台结构优化

```
[今日策略Banner — 可点击，显示当前策略效果]
[进行中订单 — 进度条 + 当前阶段 + 冲突状态]
[可接取订单 — 卡片式，包含类型/标题/奖励/倒计时]
  — 热点单：显示热度倒计时（红色紧迫感）
  — 神秘单：特殊样式，黄色边框警示
  — 每张卡片底部：[接取]按钮
```

### 2.4 指挥中心新增「私下情报行动区」

**现有内容（保留）：**
- 老板技能卡片 ×4
- 部门状态 + 工作量条

**新增（P0优先级）：**
```
[私下情报 · 可用行动]
  — 从已解锁私下频道提取的关键情报摘要
  — 每条情报后附2个行动按钮
  — 示例：「工部在研究绕过门下省 → [强化门下省权限] [装作不知道]」
```

这是私下频道→决策的核心联动入口，解决目前「偷看有内容但无法行动」的断层。

## 三、各页面详细规格

### 3.1 消息页（主页）

#### 频道边栏规格（左侧，72px宽）
```lua
-- 公开频道组
{ id="global",    label="全局",   icon="📢", bg="#D28232" }
{ id="workflow",  label="工作流", icon="🔄", bg="#4A6BF5" }
{ id="zhongshu",  label="中书省", icon="中", bg="#7B68EE" }
{ id="gongbu",    label="工部",   icon="工", bg="#E87040" }
{ id="menxia",    label="门下省", icon="门", bg="#20A8D8" }

-- 分隔线（深色）
{ type="divider" }

-- 私下频道组（深色头像）
{ id="secret_1",  label="摸鱼群", icon="👀", bg="#2D2D3A", isSecret=true }
{ id="secret_2",  label="蛐蛐群", icon="🤪", bg="#2D2D3A", isSecret=true }
```

每个频道图标：
- 宽48px 高48px，圆角10px
- 活跃状态：背景 rgba(D28232, 0.25)，左侧橙色竖条 2px
- 未读红点：右上角，≥1条显示

#### 消息气泡规格
| 类型 | 样式 |
|------|------|
| AI部门消息 | 白色气泡，左上角方形（0 10 10 10），部门颜色头像 |
| 老板消息 | 橙色气泡，右侧，圆角 10 0 10 10 |
| 系统通知 | 居中灰色胶囊，12px |
| 奇观事故 | 黄红渐变背景卡片，标题加粗，独立卡片样式 |
| 冲突消息 | 红色左边框，浅红背景 |

#### 警报横幅（Alert Bar）
- 仅在以下情况显示：工作流冲突、事故触发、验收失败
- 红色背景 #FEE8E8，字色 #E84040
- 点击可跳转到相关频道

### 3.2 工作台页

#### 订单卡片规格

```
[类型标签] [订单标题] [倒计时(热点单专属)]
[参与部门] · [难度星级]
[奖励金额]  [高品质+20%提示]
[接取按钮 — 全宽，橙色]
```

订单类型配色：
- 热点 🔥：红色 #E84040 背景 #FEE8E8
- 品牌 💎：紫色 #7B68EE 背景 #F0EEF8
- 应用 📱：蓝色 #4A6BF5 背景 #EEF1FE
- 神秘 ❓：黄色 #F0A020 背景 #FEF5E0，**加黄色边框**

进行中订单状态条：
```
接单 → 执行 → 审查 → 验收 → 结算
（当前阶段高亮，冲突时红色闪烁）
```

### 3.3 指挥中心页

#### 技能卡片规格
```
[图标 22px]
[技能名 12px 粗]
[效果描述 10px 灰]
[消耗/代价 10px 红]
[冷却天数 badge — 右上角灰色胶囊]
```

状态：
- `available`：橙色边框 1.5px
- `cooldown`：整体 opacity 0.5，禁止点击
- `no_money`：红色边框，底部显示「资金不足」

#### 部门状态规格
```
[部门彩色头像] [部门名称] [当前任务简介]
               [工作量进度条]
               [状态标签：正常/高压/停摆]
[状态图标 → 右侧]
```

工作量颜色：
- 0~60%：部门主色
- 60~80%：黄色 #F0A020
- 80%+：红色 #E84040，附⚠图标

#### 私下情报行动区（新增）
```lua

  summary = "工部已掌握绕过门下省的关键词规律",

    { label="装作不知道",     skill=nil }
  }
}
```

仅当对应私下频道已解锁且当天有新内容时显示。

### 3.8 验收结果页面规格（新增全屏模态）

当前验收结果只在聊天里显示一条消息，缺乏仪式感。  
改为触发全屏结果页：

```
[顶部：评分大字 + 验收官性格名称 + 通过/失败状态]

[验收官点评：引用样式，12px]

[逐项评分列表：每条含分数Pill + 评分维度]

[结算区：基础奖励 + 加成计算 + 声誉变化]

[返回工作台 按钮]
```

4种验收官性格（对应不同 system prompt，独立区分）：
| 订单类型 | 验收官角色 | 关注维度 |
|---------|----------|---------|
| 热点单 | 急性子网红编辑 | 爆点/前3秒钩子/传播性 |
| 品牌单 | 挑剔品牌总监 | 调性/一致性/视觉化 |
| 应用单 | 务实产品经理 | 功能完整性/可用性 |
| 神秘单 | 身份不明甲方 | 随机维度，评标模糊 |

## 四、事故/奇观卡片规格（新增）

事故触发时在工作流频道插入特殊样式卡片：

```
[卡片背景：黄红渐变]
[事故等级标签：轻/中/重/奇观]
[事故名称 — 粗体 13px]
[事故描述 — 12px，1-2行]
[当前影响 — 列表格式]
[（可选）老板干预按钮]
```

奇观级事故应该有截图价值——配色夸张、描述幽默、带公司名水印。  
建议增加到 **至少10个**（每种组织原型至少2个专属）：

| 奇观事故名 | 触发条件 | 效果 |
|-----------|---------|------|
| 审批无限循环 | 中书省门下省互相打回≥3次 | 工部停摆 |
| 门下省自我攻击 | 门下省权限>80%且无人干活 | 把自己的汇报也打回了 |
| 中书省政变 | 中书省拍板>2次 | 私自调整执行方向 |
| 热点已死 | 热点单审批>45s | 话题已凉，验收必失败 |
| 全员讨论老板 | 拍板>3次 | 今日所有部门效率-30% |
| 空转停机 | 工部满载+门下省打回 | 今日无法接新单 |

## 六、关键交互规范

### 6.1 消息发送节奏
- 同一部门连续消息合并到同一头像下（类微信气泡合并）
- AI消息推送间隔：0.8~2.5秒（模拟真实打字感）
- 事故/冲突消息：特殊动画（轻微抖动）

### 6.2 频道切换
- 点击左侧频道图标 → 立即切换，动画0.15s
- 切换到私下频道 → 整体UI变暗色主题
- 切换回公开频道 → 恢复浅色主题

### 6.3 技能使用反馈
- 使用「强制拍板」：在工作流频道插入系统消息「[老板] 强制推进执行，门下省已备案风险」
- 使用「临时外包」：在工作流频道出现新AI角色头像，背景色不同于三省

### 6.4 今日策略入口
- 状态栏策略chip：左轻触 → 弹出底部sheet（策略选择）
- 工作台顶部banner：点击 → 跳转策略全屏页

## 七、技术实施说明（Lua/UrhoX）

### 频道边栏实现
```lua
-- ChannelListPanel.lua 修改方向
-- 从右侧小点改为左侧72px竖向排列
-- 每个频道使用 DEPT_BADGE_COLORS 对应颜色
-- 私下频道分组用深色背景渲染
```

### 验收官4种性格
```lua
-- AcceptanceAgent.lua 新增
local ACCEPTANCE_PROMPTS = {
  hotspot = "你是急性子的网红编辑小磊，只在意爆点和前3秒钩子...",
  brand   = "你是挑剔的品牌总监陈总，极度在意调性和一致性...",
  app     = "你是务实的产品经理，只看能不能用，文案质量无所谓...",
  mystery = "你是身份不明的甲方，评判标准模糊，倾向随机给分..."
}

function AcceptanceAgent.GetSystemPrompt(orderType)
  return ACCEPTANCE_PROMPTS[orderType] or ACCEPTANCE_PROMPTS["brand"]
end
```
